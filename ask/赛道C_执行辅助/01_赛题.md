
# 赛题方向C-执行辅助 Agent
## 企业场景
企业X在参与各类平台(如天猫、淘宝、京东 etc.)活动或内部营销项目时，往往需要按照一系列活动规则与限制条件执行选品与落地操作。相关规则通常以文本形式存在，内容复杂且细节多，执行过程高度依赖人工解读与经验判断。在实际操作中，规则理解偏差或执行遗漏，容易导致活动效果不达预期，甚至引发后续返工。

## 赛题任务
请构建一个执行辅助 Agent，能够解析活动规则文本，将其转化为结构化的执行检查清单和决策流程，辅助运营人员理解和执行复杂规则。
Agent 的定位重点是：解析规则、生成检查清单、提示风险点，并标注需要人工确认或补充澄清的信息。

说明：活动规则示例（简化版）：
天猫 618 大促选品规则：
1. 参与商品必须满足：近30天销量≥100件；好评率≥95%；库存≥500件。
2. 价格要求：活动价不得高于近30天最低价；折扣力度≥7折。
3. 品类限制：黄金类目单店最多5个 SKU；钻石类目单店最多10个 SKU。
4. 互斥规则：已参加“品牌日”活动的商品不可重复报名。

## 需求提炼

### 企业场景
参与天猫、淘宝、京东等平台活动时，需按活动规则执行选品与落地操作。规则以文本形式存在，内容复杂、细节多，执行高度依赖人工解读与经验判断。

### 核心痛点
| 痛点 | 场景描述 |
| :--- | :--- |
| 规则理解依赖人工经验 | 需自行拆解复杂规则，易遗漏关键限制条件 |
| 从规则到执行缺乏桥接 | 文字要求难快速转化为可执行判断路径 |
| 执行风险不可控 | 信息不足时难以及时识别潜在风险点 |

### 赛题任务
构建执行辅助 Agent，解析活动规则文本，转化为结构化执行检查清单和决策流程。Agent 的定位重点是：解析规则、生成检查清单、提示风险点，并标注需要人工确认或补充澄清的信息。

### 天猫618规则示例（简化）
- 近30天销量 ≥ 100件 · 好评率 ≥ 95% · 库存 ≥ 500件
- 活动价 ≤ 近30天最低价 · 折扣力度 ≥ 7折
- 黄金类目单店最多5个SKU · 钻石类目最多10个
- 已参加"品牌日"的商品不可重复报名

> 参赛者可围绕规则解析方式、执行路径设计、不确定性处理以及人机协同机制进行探索，提出有助于提升执行效率与稳定性的创新方案。
