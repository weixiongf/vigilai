
# 赛题方向B-协作沟通 Agent
## 企业场景
企业X在日常电商运营与活动筹备过程中，需要对「主推商品」及「活动参与品类」作出快速决策。相关讨论通常涉及运营、商品、市场等多个角色，不同岗位对于决策重点的关注角度存在明显差异。由于缺乏统一的决策表达方式和共识形成机制，同一选品或主推讨论往往通过多轮沟通反复确认，沟通成本高，且容易出现理解偏差或执行口径不一致的问题。

## 赛题任务
请设计一个 协作沟通 Agent，用于支持跨角色团队在选品或主推决策过程中的讨论与共识形成。
该Agent应能够根据用户输入的选品提案文本（包含选品理由、市场观察、业务假设等），从不同业务角色的视角进行结构化解读，识别各角色可能的关注点和潜在分歧，帮助团队形成结构化的讨论框架。

说明：Agent需要基于业务常识和角色特征，推理各角色可能的关注点。
例如：用户输入“建议主推A款项链，因为小红书声量高”，Agent需要识别出市场角色可能关注“品牌定位是否匹配”，财务角色可能关注“ROI预测”，运营角色可能关注“库存与活动节奏是否匹配”等。

## 需求提炼

### 企业场景
电商运营与活动筹备中，主推商品及参与品类决策涉及运营、商品、市场等多角色，缺乏统一决策表达方式，同一建议往往经多轮反复确认，沟通成本高。

### 核心痛点
| 痛点 | 场景描述 |
| :--- | :--- |
| 跨角色视角不一致 | 不同部门关注重点各异，建议易被推翻 |
| 共识形成效率低 | 依赖会议沟通，难以快速聚焦分歧 |
| 信息表达缺乏结构 | 以经验性描述为主，缺统一框架难复用 |

### 赛题任务
设计协作沟通 Agent，根据选品提案文本（包含选品理由、市场观察、业务假设等），从不同业务角色视角进行结构化解读，识别各角色可能的关注点和潜在分歧，帮助团队形成结构化讨论框架。Agent 需基于业务常识和角色特征推理各角色可能的关注点。

### 示例
- 输入：「建议主推A款项链，因为小红书声量高」
- 市场角色 → 品牌定位是否匹配
- 财务角色 → ROI 预测
- 运营角色 → 库存与活动节奏是否匹配

> 参赛者可从跨角色视角建模、决策表达结构、分歧识别与协作提示等角度进行创新设计，探索如何通过 AI 提升团队沟通效率与决策一致性。
