## 1. 赛题背景与目标

### 1.1 企业场景
企业X是一个业务覆盖多个海外市场的品牌型企业，在不同国家及地区同时面对快速变化的市场环境，包括竞争对手动态、产品趋势、社交媒体声量、销售平台策略调整以及相关法律与合规要求的变动。由于市场资讯来源高度分散且更新频率高，企业需要在不依赖内部数据的前提下，快速掌握各市场的关键变化，并以一致且可对比的方式，汇总为管理层与营运团队可快速理解与使用的决策参考内容。

### 1.2 赛题任务
设计并构建一个**海外市场战略情报Agent**，在不接入任何企业内部数据的前提下，能够主动从多类型公开信息来源（如新闻网站、社交媒体、行业报告、电商平台公告等）获取并整合市场动态信息，对不同海外市场的变化进行结构化分析与归纳，并以“每日战略简报”的形式输出，支持运营与管理团队进行快速判断与决策参考。

### 1.3 输出要求
- 不同市场的关键变化要点（竞争、产品、平台、社媒、法规等）
- 对业务潜在影响的判断（机会 / 风险 / 需关注事项）
- 建议的观察重点或后续行动方向
- 鼓励从信息筛选逻辑、分析框架、输出结构、可追溯性及人机协同等角度进行探索

## 2. 系统总体设计

### 2.1 架构概览
```
┌──────────────────────────────────────────────┐
│                   前端展示层                    │
│       驾驶舱 / 简报 / 市场动态 / 采集调度        │
├──────────────────────────────────────────────┤
│                  Django 应用层                 │
│   用户管理 · 数据接口 · 报告生成 · LLM调度      │
├──────────────┬──────────────┬────────────────┤
│   任务队列    │    缓存层     │    数据库层     │
│   Celery +   │    Redis     │  PostgreSQL    │
│  Redis代理   │  缓存/会话   │   持久化存储    │
└──────────────┴──────────────┴────────────────┘
        ▲              ▲              ▲
        │ 爬虫调度     │ 实时推送     │ 元数据管理
┌───────┴──────┐ ┌───┴────┐ ┌─────┴──────────┐
│  Scrapy 集群  │ │ WebSocket│ │  LLM 供应商 API│
│  + 爬虫状态表  │ │ 实时推送 │ │ (OpenAI等)     │
└──────────────┘ └────────┘ └──────────────┘
```

**分层职责**  
- **PostgreSQL**：唯一数据真相源，存储所有结构化数据  
- **Redis**：高性能缓存、Celery任务队列、WebSocket通道、实时计数  
- **Django**：业务逻辑中枢，管理ORM、用户、权限、API、看板  
- **Celery**：异步任务引擎，处理爬虫调度、数据分析、LLM调用、报告生成、通知发送  

### 2.2 核心数据流
```
信息源 → Scrapy/API采集 → 原始信息(RawInfo) → Celery触发LLM分析
                                             ↓
                    ┌──→ 维度分类 + 价值评分 ──→ 更新RawInfo字段
                    ├──→ 高影响即时通知 ──→ 钉钉/飞书/邮件
                    └──→ 每日简报聚合 ──→ AnalysisReport
```
所有分析过程保留在详情中，实现“原文→分析→简报”完整追溯链。

## 4. 站点框架与后端设计

### 4.1 技术栈

| 组件 | 方案 | 说明 |
|------|------|------|
| Web框架 | Django 4.2+ | ORM、Admin、模板、REST API |
| 数据库 | PostgreSQL 16 | 主存储 |
| 缓存/队列 | Redis 7 | Celery Broker、缓存、实时计数、WebSocket |
| 异步任务 | Celery 5.3+ | 爬虫调度、分析、通知 |
| WebSocket | Django Channels | 实时推送 |
| REST API | Django REST Framework | 可选（前后端分离时使用） |
| 前端 | Django Templates + HTMX/Alpine.js | 轻量看板（可后期升级为Vue） |
| 部署 | Docker Compose | 开发/测试环境 |

### 4.2 核心数据模型

**信息源 (InfoSource)**  
- 字段：name, url, source_type(api/web/rss/social), priority(critical/high/medium/low), spider_name, crawl_frequency, last_crawled_at, last_status, is_active

**原始信息 (RawInfo)**  
- 通用字段：title, content, summary, url(unique), published_at, fetched_at, language, raw_json  
- 战略分析字段：strategic_dimension(竞争/产品/平台/社媒/法规), target_market, country, impact_type(机会/风险/需关注), severity(高/中/低), impact_score, sentiment, reference_quote  
- 元数据：is_simulated, source  

**贵金属价格 (MetalPrice)** — 可选保留，作为宏观经济指标的一部分  
- 字段：metal(XAU/XAG/XPT/XPD), price, currency, recorded_at  

**爬取任务 (CrawlJob)**  
- 字段：source, started_at, finished_at, items_fetched, items_new, status, error_log  

**分析报告 (AnalysisReport)**  
- 字段：trigger, period_start, period_end, summary, detailed_analysis, recommended_actions, affected_markets, generated_at, llm_provider  

**通知记录 (Notification)**  
- 字段：recipient, notify_type(im/email/sms/webhook), title, content, sent_at, status, reference_info(FK to RawInfo)  

**用户反馈 (UserFeedback)** — 人机协同  
- 字段：raw_info(FK), action(confirmed/corrected/ignored), correction_note, created_by, created_at  

### 4.3 Celery任务体系

| 任务 | 说明 | 调度 |
|------|------|------|
| `trigger_crawl(source_id)` | 单个信息源爬取 | 由beat根据频率触发或手动 |
| `process_raw_info(raw_info_id)` | 异步LLM分析（维度、评分） | 信息入库后自动触发 |
| `send_high_impact_notification(info_id)` | 高影响即时通知 | 评分>=阈值时自动触发 |
| `generate_daily_brief()` | 生成每日战略简报 | Celery Beat (每天9:00) |
| `retry_failed_notifications()` | 重试失败通知 | 定时 |

**Beat调度配置示例**  
```python
app.conf.beat_schedule = {
    'daily-brief-9am': {
        'task': 'tasks.generate_daily_brief',
        'schedule': crontab(hour=9, minute=0),
    },
    'check-due-spiders': {
        'task': 'tasks.trigger_all_due_spiders',
        'schedule': 60.0,  # 每分钟检查
    },
}
```

### 4.4 LLM分析流水线

1. **预处理**：语言检测、实体抽取（使用LLM或轻量NLP）  
2. **维度分类**：竞争/产品/平台/社媒/法规  
3. **市场识别**：目标市场（北美/欧洲/东南亚…）+ 国家  
4. **影响判断**：机会/风险/需关注  
5. **价值评分**（0-10分）  
   - 业务相关度 (0-4)  
   - 时效紧迫度 (0-3)  
   - 信息权威性 (0-2)  
   - 影响规模 (0-1)  
6. **摘要与建议**：≤100字摘要 + 可执行建议  
7. **简报整合**：按市场、维度、等级聚合  

评分与等级映射：  
- ≥8：高价值紧急 → 即时通知 + 简报头条  
- 5-7：中等价值 → 简报正文  
- 3-4：低价值 → 归档，周末提及  
- <3：噪音 → 丢弃/标记

## 5. 仿真数据方案（双轨备用）

### 5.1 仿真需求
- 开发解耦：爬虫与上层分析可并行开发  
- 抗风险：爬虫不可用时系统照常运行  
- 演示友好：黑客松现场稳定展示  
- 压力测试：可生成万级数据  

### 5.2 数据量规划

| 层级 | 数量 | 用途 |
|------|------|------|
| 种子数据 | 200-500条 | 覆盖所有市场/维度/影响类型，手工精修，演示用 |
| 日增量脚本 | 50-200条/天 | 模拟真实日度信息流 |
| 压测脚本 | 1万-10万条 | 数据库/并发测试 |

### 5.3 生成方法
使用LLM批量生成JSON数据，严格遵循RawInfo字段规范。  
**Prompt要求**：角色设定为分析师，给定市场、维度、影响类型，参考示例生成标题、内容、引述等。  
**质量控制**：校验字段完整性、枚举值、日期合理性。  

**示例JSON**：  
```json
{
  "title": "Lazada菲律宾站更新跨境商品上架规则",
  "content": "...",
  "strategic_dimension": "regulation",
  "target_market": "东南亚",
  "country": "菲律宾",
  "impact_type": "risk",
  "severity": "high",
  "reference_quote": "...",
  "is_simulated": true
}
```

- 📋 **战略简报**  
- 🌍 **市场动态**

- 🔔 **通知记录**  
- 🔧 **系统设置**  

### 6.2 页面详细设计

#### 驾驶舱（Dashboard）
- 顶部指标卡：今日信息量、高影响警报数、待处理建议  
- 近7天信息趋势柱状图  
- 紧急-重要四象限气泡图（可交互）  
- 市场动态热力图  
- 最新简报摘要（点击跳转简报页）  

#### 战略简报
- 左侧：按日期排列的历史简报列表  
- 右侧：当日简报全文  
  - 概述  
  - 高风险条目（可折叠，点击展开分析过程/流式思考）  
  - 需关注条目  
  - 趋势预警  
  - 每条信息底部有 ✅确认 / 🔄修正 / ❌忽略 按钮  

#### 市场动态
- 筛选栏：市场、维度、影响等级、时间范围、来源  
- 卡片/表格视图切换  
- 每条显示：标题、来源、标签、影响标签、价值评分  
- 点击进入详情页（原文+引用+分析记录+关联情报）  

#### 采集调度
- 信息源卡片列表（名称、类型、最后抓取时间、状态灯）
