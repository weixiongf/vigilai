## 第一部分：仿真数据生成方案

### 1. 为什么需要仿真数据？

- **开发解耦**：爬虫系统与上层分析系统可并行开发、独立测试。
- **抗风险**：爬虫受阻（反爬、网络、目标站点改版）时系统不瘫痪，Agent照常运行。
- **压力测试**：可生成海量数据验证系统性能与稳定性。
- **演示友好**：黑客松现场网络受限时，仿真数据可保障流畅演示。

### 2. 数据量设计：需要上万条吗？

**没必要一次生成上万条**，但需要具备**按需生成上万条的能力**。建议分三层：

| 层级 | 数量 | 用途 |
|------|------|------|
| **种子数据** | 200-500条 | 覆盖所有维度、所有市场、所有影响类型的典型案例，手工精修或半自动生成，用于演示、测试 |
| **日增量** | 50-200条/天 | 模拟单日真实信息规模，用于日常运行测试 |
| **压测数据** | 1万-10万条（脚本批量） | 数据库读写性能、分析任务并发、缓存策略验证 |

**生成原则**：质量>数量。每条仿真信息必须包含完整字段，可用LLM批量生成，也可用模板+随机组合。

### 3. 仿真信息需要模拟哪些信息源类型？

按赛题要求的五个战略维度，各维度需模拟的信息源示例：

| 维度 | 信息源类型 | 仿真示例 |
|------|----------|---------|
| **竞争** | 竞品官网新闻、PR Newswire、行业媒体 | "竞品A在东南亚市场降价15%并推出买一赠一活动" |
| **产品** | 电商平台新品上架、众筹平台、专利公告 | "北美Amazon出现同类产品B，评分4.8，主打环保材质" |
| **平台** | 电商平台政策公告、算法更新通知 | "Shopee将于7月1日起调整跨境物流费率" |
| **社媒** | Twitter/X热门话题、Reddit讨论、TikTok趋势 | "TikTok上#productX话题播放量突破1亿，用户抱怨续航问题" |
| **法规** | 政府公告、行业合规文件 | "欧盟EN 71-3标准更新，特定元素迁移限值收紧" |

此外还需模拟：
- **宏观经济数据**：美联储利率决议、各国CPI等（已有FRED等免费API，可不用模拟，但需备用）
- **行业报告**：Gartner/IDC等行业报告的摘要引述

### 4. 仿真数据生成技术方案

推荐 **LLM批量生成 + 模板引擎校验** 的混合模式：

```
1. 定义数据模型（字段：标题、内容、来源、市场、维度、影响类型、严重程度、原文引述）
2. 准备 Prompt 模板，包含：
   - 角色设定：你是某海外市场行业分析师
   - 市场清单：北美/欧洲/东南亚/拉美...
   - 维度清单：竞争/产品/平台/社媒/法规
   - 影响类型：机会/风险/需关注
   - 时效性：今天、昨天、本周
   - 示例样本：提供20-30条高质量范例
3. 调用 OpenAI API / 本地模型，批量生成 JSON 格式数据
4. 后处理校验：
   - 必填字段完整性
   - 市场、维度、影响类型在枚举范围内
   - 日期合理性
   - 文本长度控制（标题<150字，内容<2000字）
5. 存储到 PostgreSQL 的 RawInfo 表，标记 is_simulated=True
```

### 5. 仿真数据的必要性结论

- 种子数据 **必须** 有，500条精修案例即可覆盖所有场景。
- 每日增量生成脚本 **必须** 有，用于系统联调和跑通完整链路。
- 压测批量生成 **建议** 有，但不是比赛重点，可在代码中预留脚本接口。


## 第二部分：信息分析、评价与报告体系

### 1. 拿到数据后分析什么？

无论是爬虫真实数据还是仿真数据，统一进入分析流水线：

```
原始信息 → 实体抽取 → 维度分类 → 市场归属 → 影响判断 → 紧急度评分 → 聚合简报
```

每一步均由 **LLM Agent** 完成（可混合规则和ML），分析流程如下：

| 步骤 | 分析内容 | 产出 |
|------|---------|------|
| 实体抽取 | 公司名、产品名、政策名、地区 | 标签列表 |
| 维度分类 | 竞争/产品/平台/社媒/法规 | 单一维度（主）+ 可多标签 |
| 市场归属 | 北美/欧洲/东南亚/中东/拉美等 | 目标市场+国家 |
| 影响判断 | 机会/风险/需关注 | 影响类型 |
| 紧急度评分 | 综合时效、影响范围、业务关联度、信息权威性 | 1-5分 |
| 摘要生成 | ≤100字管理层可读摘要 | 摘要文本 |
| 行动建议 | 1-2条可执行建议 | 建议文本 |

### 2. 评价标准：什么样的信息是有价值的？

设计一个**信息价值评分模型**，总分10分，由四个维度加权组成：

| 评估维度 | 权重 | 评分标准 | 示例 |
|---------|:---:|---------|------|
| **业务相关度** | 4分 | 直接影响目标市场主营产品/品类（4分），间接影响（2分），无关（0分） | 竞品在目标市场发布同类新品→4分 |
| **时效紧迫度** | 3分 | 24h内需响应（3分），本周内（2分），本月内（1分） | 法规本周生效→2分 |
| **信息权威性** | 2分 | 官方公告/政府文件（2分），主流媒体报道（1分），社媒传言（0.5分） | 美国FDA官网公告→2分 |
| **影响规模** | 1分 | 全市场影响（1分），单一国家（0.5分） | 欧盟政策→1分 |

**价值分级**：

| 分值 | 等级 | 处理方式 |
|:---:|------|---------|
| 8-10分 | 🔴 高价值紧急 | 即时通知管理层，加入简报头条 |
| 5-7分 | 🟡 中等价值 | 加入每日简报正文 |
| 3-4分 | 🔵 低价值 | 归档，周末汇总提及 |
| <3分 | ⚪ 噪音 | 自动丢弃或标记为“仅供参考” |

### 3. 紧急/重要/不紧急/不重要的四象限映射

结合价值评分与影响类型，自动归类：

```
                    紧急（时效<48h）
                         │
         🔴 紧急且重要      │      🟠 紧急不重要
        (价值≥8,风险)       │     (价值5-7,机会)
                         │
 重要 ──────────────────┼────────────────── 不重要
                         │
         🔵 重要不紧急     │      ⚪ 不重要不紧急
        (价值≥8,长期趋势)  │     (价值<5)
                         │
                    不紧急
```

#### 4.3 详情追溯（点击任意条目）

点击上面任何一条，可打开详情页，展示：

### 5. 人机协同反馈闭环（黑客松加分项）

在简报的每条信息旁边，为管理者提供三个按钮：

| 按钮 | 动作 | 效果 |
|------|------|------|
| ✅ **确认** | 该信息分析正确 | 加入知识库，影响评分模型微调 |
| 🔄 **修正** | 输入修正意见 | 记录人工标注，下次同类信息自动校准 |
| ❌ **忽略** | 信息不相关或误判 | 降低相似信息的价值评分权重 |

修正反馈可回写数据库，形成持续优化的强化学习环。

## 总结：你现在的行动清单

1. **仿真数据**：按上述JSON模板，用LLM生成500条种子数据，覆盖所有市场×维度×影响类型组合。
2. **分析模型**：在Django中实现`InfoEvaluator`服务，调用LLM完成维度分类+价值评分。
3. **简报生成**：实现`DailyBriefGenerator`，按上述模板渲染Markdown/HTML。
4. **人机协同**：在简报前端添加确认/修正/忽略按钮，后端记录反馈日志。

这样你们在黑客松上既能展示**真实数据爬取能力**（爬虫系统），又能展示**完整智能分析闭环**（仿真数据+分析框架），双重保障。
