## 开发实施路线建议

### 一、按获取难度分组开发

**第一组（优先开发，技术难度最低）：直接调用免费API ⭐**

共 19 个信息源，全部通过调用官方免费API即可接入，无需编写爬虫：
- FRED、BLS、BEA、ECB、Eurostat、CFTC COT、DBnomics、NewsAPI、World Bank、IMF、USGS Science Data Catalog
- CME FedWatch（需注册 + OAuth认证）
- BullionVault XML API
- Reddit API
- SPDR、SLV（官网直接下载CSV）
- 世界黄金协会GOLDHUB（直接下载CSV/Excel）

**开发工作量估算**：3-5人天（含API注册、OAuth配置、数据解析模板）

**第二组（标准难度）：编写轻量级网页爬虫 ⭐⭐**

共 11 个信息源，公开页面无复杂反爬机制，标准Python Requests + BeautifulSoup即可：
- 美联储FOMC页面、日本央行、英格兰银行、SHFE公告、Tradingster
- WPIC报告页、ISM摘要页、USGS页面
- 东方财富行情页

**开发工作量估算**：5-8人天（含页面结构解析、异常处理、定时调度）

**第三组（中等难度）：有JS渲染或轻量反爬 ⭐⭐⭐**

共 7 个信息源，需使用Selenium/Playwright或模拟请求：
- 中国人民银行（JS重定向反爬）
- 上海黄金交易所（公告和历史行情页面有反爬）
- 中国国家统计局
- Kitco、Mining.com、CNBC
- 同花顺、东方财富股吧

**开发工作量估算**：8-12人天（含反爬处理、验证码、Cookie维护）

**第四组（高难度，可后续迭代）：复杂反爬机制 ⭐⭐⭐⭐+**

共 3 个信息源，反爬防护严密：
- Investing.com（Cloudflare防护）
- 雪球（阿里云WAF + Cookie验证 + md5__1038加密 + TLS指纹检测）
- TradingView（动态JS渲染，高难度）

**建议策略**：此批次可放在系统第二期开发，第一期可先用NewsAPI等替代方案覆盖新闻类需求。

### 三、全局技术规范建议

| 项目 | 建议 |
|------|------|
| **统一调度框架** | 建议使用 APScheduler（Python）或 Airflow 实现定时调度 |
| **API调用重试机制** | 统一实现指数退避重试（exponential backoff），最大重试3次 |
| **爬虫UA设置** | 使用真实浏览器User-Agent，定期轮换 |
| **IP代理策略** | 对第三组及以上的爬虫源建议配置IP代理池 |
| **数据存储格式** | 统一存储为JSON/CSV，按信息源+日期分层存储 |
| **异常通知** | 单个信息源连续失败3次时触发告警通知 |
| **合规要求** | 遵守各网站robots.txt和服务条款；仅抓取公开数据，不绕过认证机制 |
