
问：
我们可能要考虑使用 PEST战略分析模型：
PEST 是企业外部宏观环境分析工具，用于识别影响行业 / 企业的四大类宏观力量，为战略制定提供依据：
P（Political）政治法律
E（Economic）经济
S（Social）社会文化
T（Technological）技术

---

有了数据源之后，要对获取到的信息进行分析：
确定范围：明确分析对象（某行业 / 某企业 / 某区域市场）。
收集信息：按 P/E/S/T 四维度搜集近 3–5 年数据与趋势。
筛选关键因素：挑出影响最大、变化最快的 3–5 个因素 / 维度。
评估影响：判断每个因素是机会（O）还是威胁（T），标注影响程度（高 / 中 / 低）。
结合 SWOT：把 PEST 结论导入 SWOT，匹配内部优势 / 劣势，形成 SO/ST/WO/WT 策略。

看看怎么把这些标准化的模型和分析手段融入到我们的项目中去，或者寻找更高级更科学的处理方法。

---

## 一、现有分析维度与 PEST 的映射关系

你们的系统已经设计了五大战略维度（竞争、产品、平台、社媒、法规），它们与 PEST 模型的对应关系如下：

| 现有维度 | PEST 归类 | 说明 |
|----------|-----------|------|
| 法规 (Regulation) | **P**olitical | 政府政策、行业法规、贸易协定、税收政策 |
| 竞争 (Competition) | **E**conomic | 竞争对手定价、市场份额、行业景气度 |
| 平台 (Platform) | **E**conomic | 电商平台政策、物流成本、平台费率 |
| 产品 (Product) | **T**echnological | 新技术、专利、产品迭代趋势 |
| 社媒 (Social) | **S**ocial | 消费者舆论、文化趋势、社会价值观 |
| （需新增）宏观经济 | **E**conomic | GDP、CPI、汇率、利率、大宗商品价格 |

**结论**：当前维度已自然覆盖 PEST 的四个象限，只需在**数据标签层**增加 PEST 归属标记，无需改动爬虫体系。同时在 LLM 提示词中引入 PEST 框架，让模型自动将每条信息同时打上“战略维度”和“PEST 类型”两个标签。

## 二、PEST + SWOT 集成分析流水线设计

### 2.1 整体分析架构升级

在现有分析流水线中新增两层：

```
原始信息 → 战略维度分类 + PEST标记 → 影响评估(机会/威胁) → SWOT聚合 → 策略生成
```

**新增分析步骤：**

1. **PEST 归类**：每条信息在已有五维标签基础上，自动映射到 P/E/S/T 之一（可多标签）。
2. **机会/威胁判断**：基于 LLM 分析，判断该信息对目标企业而言是**机会**还是**威胁**，并给出影响程度。
3. **周期性 SWOT 聚合**：每日/每周将 PEST 分析结果汇总，结合企业内部设定的优势/劣势（可在系统配置中预设），生成动态 SWOT 矩阵。
4. **策略建议生成**：根据 SWOT 的 SO/ST/WO/WT 交叉组合，自动生成可执行的战略建议。

### 2.2 数据库模型扩展

在现有 `RawInfo` 模型中新增字段：

```

输出JSON格式：

内部优势：{config.strengths}
内部劣势：{config.weaknesses}

外部机会（本周PEST-O汇总）：
{list_of_opportunities}

外部威胁（本周PEST-T汇总）：

2. ST策略：如何利用优势规避威胁？
3. WO策略：如何改进劣势抓住机会？
4. WT策略：如何改进劣势规避威胁？

```

### 2.5 Celery 任务新增

```python
@shared_task
def pest_analysis_batch(raw_info_ids: list):
    """批量对原始信息进行PEST分析"""
    infos = RawInfo.objects.filter(id__in=raw_info_ids)
    for info in infos:
        result = call_llm_pest_analysis(info)
        info.pest_type = result['pest_type']
        info.opportunity_or_threat = result['opportunity_or_threat']
        info.impact_level = result['impact_level']
        info.impact_rationale = result['rationale']
        info.save(update_fields=['pest_type', 'opportunity_or_threat', 
                                 'impact_level', 'impact_rationale'])

@shared_task
def generate_swot_report():
    """生成每日/每周SWOT分析报告"""
    yesterday = now() - timedelta(days=1)
    opportunities = RawInfo.objects.filter(
        published_at__gte=yesterday, opportunity_or_threat='O'
    ).values_list('summary', flat=True)
    threats = RawInfo.objects.filter(
        published_at__gte=yesterday, opportunity_or_threat='T'
    ).values_list('summary', flat=True)
    
    result = call_llm_swot_strategy(
        strengths=settings.COMPANY_STRENGTHS,
        weaknesses=settings.COMPANY_WEAKNESSES,
        opportunities=opportunities,
        threats=threats,
    )
    
    SWOTAnalysis.objects.create(
        period_start=yesterday,
        period_end=now(),
        strengths=settings.COMPANY_STRENGTHS,
        weaknesses=settings.COMPANY_WEAKNESSES,
        opportunities=list(opportunities),
        threats=list(threats),
        so_strategies=result['so'],
        st_strategies=result['st'],
        wo_strategies=result['wo'],
        wt_strategies=result['wt'],
    )
```

## 四、可选的更高级分析方法

如果黑客松评分偏向创新性，可在 PEST+SWOT 基础上引入以下方法（按实现难度排序）：

| 方法 | 核心逻辑 | 实现成本 | 建议 |
|------|---------|:---:|------|
| **PESTEL** | 在PEST基础上增加Environmental(环境)和Legal(法律) | 低 | 只需扩展标签枚举 |
| **情景规划** | LLM基于关键不确定因素生成3种未来情景（乐观/中性/悲观）及应对策略 | 中 | 在SWOT报告末尾追加一段 |
| **波特五力** | 自动分析：供应商议价力、买方议价力、新进入者威胁、替代品威胁、现有竞争者强度 | 中 | 单独一个分析任务，周报输出 |
| **关键成功因素(KSF)** | 从近期高影响信息中提取出现频率最高的3-5个成功因素 | 低 | 在周报趋势预警中呈现 |
| **关联网络图** | 将信息中的实体(公司、产品、政策)构建关联图，找出核心影响节点 | 高 | 可视化加分项，需图数据库 |

**黑客松建议**：优先实现 **PESTEL**（简单扩展）和 **情景规划**（LLM友好），这两个最能体现“战略辅助”的赛题定位。

## 五、总结：PEST+SWOT 集成后系统的完整数据流

```
信息采集 → 五维分类 + PEST标记
                    ↓
              机会/威胁判断 → 高影响即时通知
                    ↓
            每日PEST聚合 + 内部优势/劣势(人工维护)
                    ↓
              SWOT交叉策略生成
                    ↓
         每日战略简报（含PEST动态 + SWOT策略看板）
```

**一句话集成方案**：

> 在现有 `RawInfo` 表加三个字段（`pest_type`、`opportunity_or_threat`、`impact_level`），新增一张 `SWOTAnalysis` 表，升级两条 LLM 提示词，加两个 Celery 任务，前端加一个雷达图和一个 SWOT 矩阵页——完整 PEST+SWOT 战略分析闭环就做完了，与现有架构无缝兼容。
