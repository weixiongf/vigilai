
# 赛题方向D-风控合规 Agent
## 企业场景
企业X在日常运营中频繁涉及各类合同与合作协议的修订与更新。不同版本之间的变更内容往往需要业务与非法律背景人员共同理解与确认。
人工比对合同差异成本高，且容易忽略潜在风险条款变化，给后续执行和合规管理带来不确定性。

## 赛题任务
请设计一个 风控合规 Agent，用于辅助识别合同或协议不同版本之间的差异点，并将潜在风险以通俗、结构化的方式提示给非专业人员参考。
该Agent不需要给出合规结论，而应聚焦于变化识别、风险类型归类及人工关注提示。(比赛现场会提供合同示例)

说明：合同差异示例
V1版本第3条：乙方应在收到货物后7个工作日内完成验收，逾期视为验收合格。
V2版本第3条：乙方应在收到货物后3个工作日内完成验收，逾期视为验收合格。如发现质量问题，乙方有权拒收并要求甲方承担运费。

期望的Agent输出示例：
差异识别：验收期限从7天缩短为3天；新增质量问题处理条款。
风险提示：验收时间缩短可能导致来不及充分检查；新增的退货运费条款对我方有利，但需确认“质量问题”的判定标准。

# 需求提炼

### 企业场景
日常运营中频繁涉及合同与合作协议的修订，不同版本间变更需业务与非法律背景人员共同确认。人工比对成本高，且易忽略潜在风险条款变化。

### 核心痛点
| 痛点 | 场景描述 |
| :--- | :--- |
| 差异识别依赖人工 | 逐条阅读比对效率低，易遗漏关键变更 |
| 风险理解门槛高 | 非法律背景人员难判断变更条款的潜在影响 |
| 风险提示缺乏结构 | 缺少统一表达框架，关注重点不明确 |

### 赛题任务
设计风控合规 Agent，辅助识别合同不同版本间差异，将潜在风险以通俗、结构化方式提示非专业人员。Agent 聚焦于**变化识别与风险分类，不给出合规结论**。

### 示例
- **V1**：乙方应在收到货物后 7个工作日内完成验收，逾期视为验收合格
- **V2**：改为 3个工作日，新增质量问题可拒收并要求甲方承担运费

- **差异识别**：验收期限从7天缩短为3天；新增质量问题处理条款
- **风险提示**：验收时间缩短可能来不及充分检查，需确认“质量问题”判定标准

> 参赛者可从文本理解、风险表达方式、业务可读性及人工介入机制等角度进行设计，探索 AI 在风控合规辅助领域的应用可能性。
