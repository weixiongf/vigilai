
### 企业场景与核心痛点
#### 企业场景
业务覆盖多个海外市场，面对竞争对手动态、产品趋势、社媒声量、平台策略及法规变动。市场资讯来源高度分散、更新频繁，需在不依赖内部数据的前提下，快速掌握各市场关键变化，以一致且可对比的方式汇总为决策参考。企业期望透过 AI Agent，模拟一个具备「市场雷达」能力的战略辅助角色，支持跨市场营运决策与日常策略判断。

#### 核心痛点
| 痛点 | 具体场景 |
| :--- | :--- |
| 市场信息高度分散 | 新闻、报告、社群资料分散，人工查阅耗时易遗漏 |
| 判断依赖个人经验 | 不同负责人标准不一，难以形成可对比结论 |
| 缺乏结构化决策输入 | 资讯零散，晨会中无法快速形成明确行动方向 |
| 市场变化与决策时间差 | 法规及平台政策调整往往事后才被注意到 |

---

### 赛题任务
#### 任务目标
构建海外市场战略情报 Agent，不接入任何企业内部数据，主动从多类型公开信息来源（新闻网站、社交媒体、行业报告、电商平台公告等）获取并整合市场动态，进行结构化分析与归纳，以「每日战略简报」输出，支持管理与运营团队快速判断与决策参考。

#### 输出要求示例
- 不同市场的关键变化要点（竞争 / 产品 / 平台 / 社媒 / 法规）
- 对业务潜在影响的判断（机会 / 风险 / 需关注事项）
- 建议的观察重点或后续行动方向

#### 开放性探索方向
- **信息筛选逻辑**：如何从海量公开信息中高效筛选有价值内容
- **分析框架设计**：结构化归纳的方法论，保证跨市场可对比性
- **输出结构与人机协同**：可追溯性设计及人工介入节点

> 参赛者可聚焦于上述痛点，也可结合自身对跨市场运营与战略决策的理解，提出更具前瞻性或创新性的 Agent 设计方式。鼓励从信息筛选逻辑、分析框架、输出结构、可追溯性及人机协同等角度进行探索。
